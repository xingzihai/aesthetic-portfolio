## ✅ 最终正确方案

### 核心原理

```
点击位置 (x, y)
    ↓
创建涟漪实例，radius = 0
    ↓
动画循环：radius += expandSpeed
    ↓
在Canvas上绘制涟漪圈（从点击点向外扩散）
    ↓
radius >= maxRadius → 涟漪消失
```

### 技术要点

1. **Canvas涟漪层**：表层Canvas + 背后Canvas
2. **涟漪类**：记录点击位置、当前半径、透明度
3. **动画循环**：requestAnimationFrame 更新半径并重绘
4. **涟漪扩散**：从点击点(x,y)开始，半径从0增长到maxRadius
5. **波纹效果**：多个波纹圈（waves），形成水波效果

### 代码结构

```javascript
class Ripple {
  constructor(x, y) {
    this.x = x;           // 点击点
    this.y = y;
    this.radius = 0;      // 当前半径（从0开始）
    this.maxRadius = 300;  // 最大半径
  }
  
  update() {
    this.radius += 3;      // 扩散速度
    // 透明度衰减...
  }
  
  draw(ctx) {
    // 从 开始绘制涟漪圈
    ctx.arc(this.x, this.y, this.radius, ...);
  }
}
```

## 2026-04-08 涟漪折射系统开发失败总结

### 核心教训

**错误1：方向对，执行全错**
- 波形算法正确
- 但从未真正实现"采样背景像素"
- 一直在"涂颜色"

**错误2：Canvas背景捕获问题**
- Canvas默认是空的
- 无法直接用getImageData获取页面内容
- 需要先把DOM绘制到Canvas

**错误3：多次迭代浪费**
- 4次方案都是"涂颜色"
- 没有从根本上理解"折射"的技术实现

### 正确理解

**折射 = 像素采样 + 重排**
1. 捕获涟漪区域的背景像素
2. 根据波形计算偏移方向
3. 从偏移位置采样，绘制到当前位置
4. 不预设颜色，只采样和重排

### 下次开发要点

1. **先解决背景捕获**
   - html2canvas
   - 或SVG位移图
   
2. **验证采样逻辑**
   - 简单测试：采样后直接绘制
   - 确认能看到内容

3. **再添加波形计算**
   - 波峰向外采样
   - 波谷向内采样
